# SaaS Customer Analytics using SQL & Python

## Project Overview
This project simulates a SaaS (Software-as-a-Service) business environment by analyzing subscription-based customer data to understand revenue performance, customer behavior, and churn patterns.

The objective was to perform **end-to-end data analysis** using advanced SQL and Python to generate actionable business insights similar to real consulting analytics workflows.

---

## Business Problem
Subscription-based companies rely on recurring revenue and customer retention.  
This project answers key business questions:

- Which customer segments generate the most revenue?
- Which subscription contracts experience the highest churn?
- How is revenue distributed across customer groups?
- Which customers represent high-value retention targets?

---

## Tools & Technologies
- **SQL (SQLite)** – Data querying and KPI computation
- **Python** – Data analysis and visualization
- **Pandas** – Data manipulation
- **Matplotlib** – Business visualization
- **Jupyter Notebook** – End-to-end analysis workflow

---

## SQL Concepts Demonstrated
- Joins
- WHERE & HAVING clauses
- GROUP BY aggregations
- Common Table Expressions (CTEs)
- Window Functions (`NTILE`, `RANK`)
- Conditional Aggregation using CASE statements

---

## SaaS KPIs Analyzed
- Monthly Recurring Revenue (MRR)
- Customer Churn Rate
- Revenue Segmentation
- High-value Customer Identification

---

## Key Analytical Insights
- Month-to-month contracts showed significantly higher churn rates.
- Revenue concentration was observed among top customer quartiles.
- High-value customers demonstrated stronger retention behavior.
- Contract structure strongly influenced revenue stability.
